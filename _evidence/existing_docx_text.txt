Datasety Kaggle (bsthere/nike-global-catalogue-2026) zawierają globalny cennik produktów Nike na rok 2026, obejmujący 45 krajów i rynków zbytu. Pliki CSV reprezentują oficjalne ceny detaliczne sugerowane przez producenta (MSRP) w walutach lokalnych każdego rynku.

Licencja: Kaggle – dane scrapowane / syntetyczne do celów analitycznych.

1.2 Źródło 2 – NBP Web API (Kursy Walut)
Narodowy Bank Polski udostępnia publiczne REST API pod adresem https://api.nbp.pl/api/, zwracające historyczne i bieżące kursy walut w formatach JSON i XML. API oferuje trzy typy tabel kursowych i nie wymaga uwierzytelnienia.

Dostęp i warunki:
Base URL: https://api.nbp.pl/api/
Format odpowiedzi: JSON (domyślny) lub XML; wybierany przez parametr ?format=json lub nagłówek Accept.
Uwierzytelnienie: Brak – API publicznie dostępne.
Protokół: Wyłącznie HTTPS (od 1 sierpnia 2025 r. HTTP nie jest obsługiwany).
Archiwum historyczne: Kursy walut dostępne od 2 stycznia 2002 r. Jedno zapytanie obejmuje max. 93 dni.
Rozmiar danych: ok. 5–10 MB rocznie dla wszystkich walut w tabeli A.
Harmonogram odświeżania: Codziennie (dni robocze NBP); SSIS pobiera dane każdego ranka.

2. Proces ETL – Ekstrakcja, Transformacja, Ładowanie

Warstwa integracji realizowana jest w Microsoft SQL Server Integration Services (SSIS). Przepływ danych podzielony jest na dwa niezależne pakiety SSIS, orkiestrowane przez SQL Server Agent.

2.1 Pakiet SSIS #1 – Nike Catalogue CSV
Ekstrakcja
Flat File Source (CSV) z katalogu lokalnego lub lokalizacji sieciowej.
Kod strony UTF-8; separator: przecinek; enclosing: cudzysłów.
Wczytywany jeden plik kwartalnie; schemat weryfikowany względem metadanych.

Transformacja
Data Conversion: price DECIMAL, currency_code CHAR(3), country_code CHAR(2), release_year SMALLINT.
Derived Column: Generowanie klucza zastępczego product_key (HASHBYTES SHA-256 z sku + country_code), ETL_load_date = GETDATE().
Lookup do DIM_Product: Sprawdzenie, czy SKU już istnieje – jeśli tak, przekazanie do gałęzi SCD2 Update; jeśli nie – Insert.
SCD2 (Slowly Changing Dimension): Zmiany atrybutów produktu (np. cena, colorway, subcategory) zapisywane przez zamknięcie starego rekordu (valid_to = dzień poprzedni, is_current = 0) i dodanie nowego (valid_from = dziś, is_current = 1).
Lookup do DIM_Geography: Mapowanie country_code → region zgodnie z tabelą mapowania Nike.
Error Redirect: Wiersze z null SKU lub ceną ≤ 0 przekierowane do tabeli staging.error_log z kolumną reject_reason.

Ładowanie
OLE DB Destination → staging.stg_nike_prices (tabela przejściowa, TRUNCATE przed ładowaniem).
Po walidacji: MERGE do DIM_Product oraz INSERT do FACT_Prices.
Indeksy na kolumnach łączenia przebudowywane po zakończeniu ładowania (ALTER INDEX REBUILD).

2.2 Pakiet SSIS #2 – NBP FX Rates
Ekstrakcja
Script Task (Python/C#): wywołanie HTTPS GET na endpoint tabeli A i C; parsowanie JSON przy użyciu Newtonsoft.Json/Python.
Parametry: data_start, data_end pobierane ze zmiennej SSIS lub tabeli konfiguracyjnej.
Zapytania w 93-dniowych segmentach (limit API NBP) z automatycznym podziałem dużych zakresów dat.

Transformacja
Normalizacja JSON: Spłaszczenie tablicy rates[] do formatu tabelarycznego (table_type, effective_date, currency_code, mid/bid/ask).
Data Conversion: effective_date → DATE, mid/bid/ask → DECIMAL(10,6).
Lookup do DIM_Currency: Weryfikacja currency_code w słowniku ISO 4217; nieznane kody przekierowane do logu błędów.
Derived Column: fx_source = 'NBP', load_timestamp = GETDATE().

Ładowanie
INSERT INTO FACT_ExchangeRates ... WHERE NOT EXISTS (klucz: effective_date + currency_code + table_type) – deduplikacja.
Aktualizacja DIM_Currency.current_mid_rate po każdym przebiegu (SCD1).

2.4 Orkiestracja (SQL Server Agent)
Job #1 – NikeETL_FX_Daily: Uruchamiany codziennie o 07:00 (dni robocze). Pobiera tabele A i C z NBP za poprzedni dzień roboczy.

Hierarchia DIM_Category: Category → Subcategory (2 poziomy; umożliwia drill-down w Power BI).

5. Koncepcja Raportowania (Power BI)

Raporty Power BI łączą się z hurtownią danych przez tryb Import (odświeżanie raz dziennie po zakończeniu ETL). Idealnie trzy strony raportu (tabs):

5.1 Strona 1 – Global Price Map
Mapa choropleth (geograficzna) pokazująca cenę modelu wybranego przez użytkownika w przeliczeniu na PLN dla każdego z 45 krajów.
Filtry: kategoria produktu, model, data (suwak czasowy).
Tooltip: kraj, cena lokalna, waluta, kurs NBP, cena w PLN.
KPI cards: najdroższy rynek, najtańszy rynek, spread cenowy.

5.2 Strona 2 – Arbitrage Opportunities
Tabela rankingowa: TOP-N produktów z największą różnicą cenową (max-min PLN) między rynkami.
Bar chart: potencjalna marża [%] dla TOP 20 SKU.
Wykres liniowy: zmienność kursu wybranej waluty (3/6/12 miesięcy) z NBP.
Alerty warunkowe (formatowanie kolorem): marża &gt; 20% → zielony; 10–20% → żółty; &lt; 10% → czerwony.

5.3 Strona 3 – FX Dashboard
Tabela kursów wszystkich walut (Tabela A NBP) z porównaniem dnia bieżącego do średniej 30-dniowej.
Heatmapa: zmienność [%] walut na osi czasu (miesiące × waluty).
Line chart: przebieg historyczny wybranej pary walutowej.
KPI: waluty z najsilniejszym/najsłabszym trendem w ciągu ostatnich 30 dni.

Przykładowy szkic raportu (Strona 2 – Arbitrage Opportunities) dołączony jako osobny załącznik PDF.
